## Code Cohesion Report

### Summary Analysis

The project is a structural cohesion analyzer. The current architecture exhibits
a common "God Module" tendency in the scanner and a mixing of protocol and
orchestration logic in the LLM client.

### Detailed Findings

#### 1. `src/scanner.rs` - High Suspicion

Mixed responsibilities:

- domain model: defining `ScanReport`, `FileFinding`, `Role`, and `Suspicion`;
- file system orchestrator: walking directories and reading files;
- heuristic engine: mapping roles from hardcoded signals;
- analysis engine: calculating suspicion scores based on metrics;
- text processor: implementing string-literal stripping for noise reduction.

Recommendation:

- `scanner/types.rs` or `scanner/models.rs`: data structures and enums;
- `scanner/heuristics.rs`: role inference, keyword mapping, string cleaning;
- `scanner/engine.rs` or `scanner/mod.rs`: scan orchestration.

#### 2. `src/llm.rs` - High Suspicion

Mixed responsibilities:

- API client: `reqwest` headers, endpoints, JSON serialization and
  deserialization;
- tool execution: mapping LLM tool calls to `CodeReader`;
- conversation state: managing messages, tool calls, and the tool-call budget.

Recommendation:

- `llm/client.rs`: HTTP transport and OpenAI-compatible API calls;
- `llm/agent.rs`: conversation loop and tool orchestration;
- `llm/types.rs`: protocol DTOs;
- `llm/tools.rs`: tool schema and tool execution.
